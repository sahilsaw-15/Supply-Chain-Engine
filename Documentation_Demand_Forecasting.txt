STAGE 1:
Advanced Data Cleaning, Validation, and Forecasting Dataset Pipeline
This update significantly expands the project from a basic CSV reader into a structured Java-based Supply Chain Analytics Engine capable of data ingestion, cleaning, validation, quality assessment, and forecasting dataset preparation.

Project Summary :
	- Source Records: 180,519
	- Cleaned Records: 180,519
	- Corrected/Standardized Rows: 159,889
	- Likely Duplicate Orders: 11,370
	- Forecasting Records Generated: 22,273

1. Data Ingestion Improvements
	- Implemented CSV loading for the DataCoSupplyChainDataset containing 180,519 rows and 53 columns.
	- Replaced basic comma-based parsing with quote-aware CSV parsing to correctly handle commas inside quoted values.
	- Replaced fixed column-index access with column-name based mapping, making the system resilient to future dataset column order changes.
	- Eliminated silent row skipping and added safe conversion logic with reporting.
	- Added safeguards to prevent runtime crashes when no records are available.

2. SupplyChainRecord Expansion
The SupplyChainRecord model was expanded to organize and represent additional supply chain, logistics, product, shipment, location, and financial attributes already available within the source dataset.

The record model now includes:
	- Product and category information
	- Department information
	- Customer location details
	- Order location details
	- Market information
	- Payment information
	- Shipping and delivery details
	- Order status information
	- Quantity, pricing, sales, and profit fields
	- Geographic coordinates
	- Order and shipping timestamps

3. Data Cleaning Enhancements
3.1 Header Cleaning
	- Trimmed column names
	- Detected blank headers
	- Detected duplicate headers
3.2 Row Structure Validation
	- Repaired rows with missing columns
	- Trimmed rows containing extra columns
	- Ensured consistent row structure throughout the dataset
3.3 Text Standardization
	- Removed unnecessary whitespace
	- Collapsed repeated spaces
	- Replaced missing critical text fields with UNKNOWN
3.4 Country Normalization
Standardized country names into a single English representation.
Examples:
	• EE. UU. → United States
	• US → United States
	• USA → United States
	• Japón → Japan
	• Corea del Sur → South Korea
	• Alemania → Germany
	• España → Spain
	• Países Bajos → Netherlands
	• República Democrática del Congo → Democratic Republic of the Congo
Added accent-insensitive matching to correctly process international values.

4. Numeric and Date Validation
4.1 Numeric Cleaning
Added safe parsing and validation for:
	- Sales
	- Profit
	- Product Price
	- Order Item Total
	- Discounts
	- Discount Rates
	- Profit Ratios
	- Sales Per Customer
	- Shipping Days
	- Quantities
	- IDs and status fields

Invalid values are safely converted to default values rather than causing failures.

4.2 Date Cleaning
	- Parsed Order Date using Java LocalDateTime
	- Parsed Shipping Date using Java LocalDateTime
	- Validated chronological consistency between dates
	- Verified shipping duration against actual date differences

5. Validation and Quality Checks
5.1 Duplicate Detection
	- Duplicate detection using Order ID + Order Item ID
	- Added deeper likely-duplicate detection using business attributes

Results:
	• Duplicate Order Items Found: 0
	• Likely Duplicate Orders Found: 11,370

5.2 Business Rule Validation
Implemented rule-based consistency checks including:
	- Delivery risk validation
	- Delivery status validation
	- Order status validation
	- Discount consistency validation
	- Profit consistency validation
	- Profit ratio validation
	- Quantity and sales consistency checks
	- Shipping timeline validation

Generated: 
	• 18,511 Business Rule Warnings

6. Statistical Outlier Detection
Implemented both threshold-based and statistical anomaly detection.
Detected:
	• Profit Outliers: 3,608
	• Sales Outliers: 467

Additional checks include:
	- Suspiciously high sales
	- Excessive profit or loss values
	- Unusual shipping durations
	- Invalid numeric ranges
	- Geographic coordinate validation

7. Privacy and Sensitive Data Handling
The following fields are intentionally excluded from analytics processing:
	- Customer Email Address
	- Customer Password
	- Customer First Name
	- Customer Last Name
	- Customer Street
	- Customer Zip Code
	- Product Image
	- Product Description
	- Order Zip Code
	
These fields are not required for demand forecasting and were removed from the analytical workflow.

8. Configurable Cleaning Framework
Introduced a configurable cleaning architecture.
Added Components
	- CleaningConfig
		• Centralized thresholds and validation rules
		• Reduces hardcoded values
	- CleanedDataExporter
		• Exports cleaned datasets to Excel-friendly CSV files
	- SaveLocationChooser
		• Supports custom output locations
	- Lightweight Test Suite
		• Validates cleaning logic without requiring JUnit

9. Dual-Output Data Pipeline
One of the major improvements in this update is the creation of a multi-stage data pipeline that generates two datasets from a single raw source.

Dataset 1: Cleaned Order-Level Dataset
Generated File:
Data/cleaned_supply_chain_data.csv
Purpose:
	- Preserves individual order records
	- Stores cleaned and validated operational data
	- Serves as the master cleaned dataset
	- Can be reused for future analysis and auditing
	
Dataset 2: Forecasting-Ready Demand Dataset
Generated File:
Data/forecasting_demand_dataset.csv

Purpose:
	- Converts transactional order data into a forecasting-ready time-series demand dataset.
	- Aggregates records by:
		• Order Date
		• Product Name
		• Category Name

Generated Features:
		- Order Date
		- Product Name
		- Category Name
		- Total Quantity
		- Total Sales
		- Total Profit
		- Order Count

Transformation Results:
		• Original Order Records: 180,519
		• Forecasting Demand Records: 22,273

This significantly reduces transaction-level noise and creates a structured time-series style dataset suitable for future forecasting models.

Pipeline Flow
	Raw Dataset (180,519 Rows)
	       ↓
	CSV Data Loader
	       ↓
	Data Cleaning & Validation Engine
	       ↓
	cleaned_supply_chain_data.csv
	       ↓
	Demand Dataset Builder
	       ↓
	forecasting_demand_dataset.csv
